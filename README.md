# 回声定位炮 — EchoLure

基于 **PyGame** 开发的射击小游戏，核心机制围绕「声呐探测 + 预判射击」展开。

## 游戏简介

画布中漂浮着处于**量子隐形**状态的目标，肉眼不可见。玩家需要依赖炮台自动充能的**声呐波**短暂显形目标，在窗口期内完成击杀。未显形的目标炮弹会直接穿过，无法造成伤害。

> **核心乐趣**：在有限信息下预判射击，一发反弹盲狙命中死角目标的瞬间，全场"哇"声一片。

## 操作方式

| 操作 | 说明 |
|------|------|
| **鼠标移动** | 控制炮台瞄准方向 |
| **空格** | 发射炮弹 |

> 零额外按键，所有深度都来自策略判断，不在操作复杂度。

## 核心机制

### 探测 → 瞄准 → 射击

- **声呐充能**：炮台每 **4秒** 自动向鼠标方向发射一道窄扇形声呐波，扫到的目标显形 **2.5秒**
- **充能环**：炮台下方的环形进度条像呼吸灯一样收缩，归零时自动发射声呐
- **受惊反应**：被声呐扫到的目标会在 **0.5秒** 内轻微加速并随机偏转，不能完全依赖箭头预判
- **炮弹反弹**：炮弹可在画布边缘反弹 **1次**，利用反弹可击杀视觉死角里的目标

### 道具系统（触碰金色光球自动拾取，最多持1个）

| 道具 | 按键 | 效果 |
|------|------|------|
| **全频扫描** | W | 下一次声呐变为 **360°环形扫描**，全屏目标显形 **4秒** |
| **黏着信标** | S | 下一次炮弹命中后标记目标 **6秒** 持续显形 |
| **跳弹散芯** | R | 下一次炮弹在**首次反弹后**分裂为 **3发弹片**（30°扇形） |

> 道具都是"改变下一次XX的性质"，不需要额外按键，系统自动消耗已持有的道具。

### 计分系统

| 得分事件 | 分值 | 说明 |
|---------|------|------|
| 常规击杀 | +100 | 声呐显形后命中 |
| 盲狙击杀 | +400 | 未显形时靠反弹蒙中（极难，但爽快） |
| 窗口连击 | +50×N | 单次显形窗口期内每多杀一个，递增50 |
| 极速反应 | +50 | 显形后 **1秒内** 完成击杀 |
| 噪音惩罚 | - | 连续3发未命中，炮台进入 **2秒** 噪音干扰（声呐充能暂停） |

### 难度曲线

- 初始 **2个目标**，速度中等
- 每累计 **800分**，增加 **1个新目标**，全体目标基础速度 **+8%**
- 当目标数达到 **4个** 后，引入**静默目标**（被声呐扫到时只显示一个**闪烁的红点**，不显示轮廓和方向箭头）

## 运行方式

需要 Python >= 3.12，推荐使用 [uv](https://docs.astral.sh/uv/) 管理环境：

```bash
# 安装依赖
uv sync

# 运行游戏
uv run python main.py
```

或使用 pip：

```bash
pip install -e .
python main.py
```

## 技术栈

- **PyGame** — 2D 游戏引擎
- **NumPy** — 程序化音效生成
- **uv** — Python 包管理与环境隔离

## 开发背景

本项目为课堂演示任务，从需求构思到完整实现均由 AI 辅助完成：

1. **需求设计**：通过 AI 对话发散完善游戏逻辑，产出设计文档
2. **代码实现**：由 AI 工具（OpenCode）分步执行开发，包括架构设计、核心系统实现、道具系统、视觉效果等
3. **迭代优化**：测试发现问题后持续修复（字体适配、模块分离、道具触发反馈等）
